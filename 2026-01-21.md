# Azure Sentinel Updates: Last 12 Months (January 2025 - January 2026)

## Overview
Over the past year, Microsoft Sentinel has undergone significant evolution, transforming from a traditional SIEM into a comprehensive security platform with AI-driven capabilities and enhanced detection frameworks. This document outlines the major changes and new features introduced.

---

## Major Strategic Shifts

### Platform Evolution: SIEM to AI-Driven Platform (September 2025)
Microsoft Sentinel evolved beyond traditional SIEM functionality to become a comprehensive security platform designed for the AI era. This shift includes:
- **Platform-led, machine-speed defense** replacing static, rule-based controls
- **Unified data architecture** for agentic defense
- **Three key architectural enhancements**

## Key Features Released (Last 12 Months)

### 1. Microsoft Sentinel Data Lake - Generally Available (September 2025)
- **Purpose**: Scalable, cost-efficient foundation for long-term data retention and multi-modal analytics
- **Benefits**:
  - Unified security data across sources
  - Advanced analytics without infrastructure overhead
  - Long-term retention capabilities
  - Cost optimization compared to traditional approaches

### 2. Microsoft Sentinel Graph - Preview (September 2025)
- **Purpose**: Unified graph analytics for deeper context and threat reasoning
- **Capabilities**:
  - Models relationships across users, devices, and activities
  - Supports complex threat investigations
  - Enables pre- and post-breach analysis
  - Provides contextual relationship mapping

### 3. Microsoft Sentinel Model Context Protocol (MCP) Server - Preview (September 2025)
- **Purpose**: Hosted interface for building intelligent agents using natural language
- **Benefits**:
  - Simplifies agent creation and data exploration
  - Allows querying and reasoning over security data
  - Eliminates need for schema knowledge
  - Natural language query interface

### 4. Enhanced User and Entity Behavior Analytics (UEBA) - Expanded (2025)

#### New Data Sources for UEBA Anomaly Detection (September 2025)
Six new data sources now integrated:

**Microsoft Authentication Sources:**
- **Microsoft Defender XDR device logon events**: Captures endpoint logon activity for lateral movement detection
- **Microsoft Entra ID managed identity signin logs**: Tracks service identity sign-ins for script and service monitoring
- **Microsoft Entra ID service principal signin logs**: Monitors app and script service principals

**Third-Party Cloud Platforms:**
- **AWS CloudTrail login events**: Detects risky AWS logins (failed MFA, root account usage)
- **GCP audit logs**: Captures denied IAM access attempts and privilege escalation attempts
- **Okta authentication events**: Monitors MFA challenges and authentication policy changes

#### UEBA Behaviors Layer - Preview (January 2026)
A revolutionary addition transforming raw security logs into actionable behavioral insights:
- **Real-time aggregation**: Converts high-volume, low-level logs into human-readable behaviors
- **AI-powered normalization**: Sequences raw events into meaningful behavior patterns
- **MITRE ATT&CK mapping**: Provides context on tactics, techniques, and procedures (TTPs)
- **Current support**: AWS CloudTrail, CommonSecurityLog (CyberArk, Palo Alto), GCP Audit Logs
- **Key benefits**:
  - Accelerates investigations through behavior aggregation
  - Transforms noisy telemetry into clear insights
  - Works independently from UEBA anomaly detection
  - Supports all SOC personas (analysts, hunters, detection engineers)

#### UEBA in Defender Portal - Preview (November 2025)
- **Anomaly tagging**: Users with behavioral anomalies tagged as "UEBA Anomalies"
- **Top anomalies view**: Displays top 3 anomalies from past 30 days
- **Drilldown capability**: Quick access to all user anomalies from incident graphs
- **Enhanced hunting**: Contextual banners prompt queries to include UEBA anomaly data

### 5. Threat Intelligence Enhancements (October 2025)

#### STIX/TAXII Export Capability - Preview
- **Bi-directional intelligence sharing**: Export threat intelligence objects back to external platforms
- **TAXII 2.1 support**: Currently available for TAXII 2.1-based platforms
- **Reduced manual processes**: Eliminates need for custom playbooks for TI distribution
- **Portal integration**: Available in both Defender and Azure portals

### 6. SAP Sentinel Solution Enhancements (2025)

#### Agentless SAP Data Connector - Generally Available (November 2025)
- **Deployment method**: Agentless architecture for simplified deployment
- **Pricing**: Same cost as previous containerized connector
- **Deprecation notice**: Containerized SAP connector support ends September 30, 2026
- **Migration required**: All deployments should migrate to agentless connector

#### Expanded SAP Detection Coverage (January 2026)
New detections across multiple SAP BTP components:
- **SAP Integration Suite**: Unauthorized artifact changes, access policy modifications
- **SAP Cloud Identity Service**: User deletion monitoring, privilege grant tracking, SAML/OIDC changes
- **SAP Build Work Zone**: Mass role deletion detection, unauthorized resource access
- **SAP BTP Audit Logging**: Audit log ingestion gap detection

### 7. Workbook Enhancements (August 2025)
- **Direct editing in Defender portal**: Create and edit workbooks without returning to Azure portal
- **Enhanced UX**: Streamlined workflow for analysts

### 8. Account Entity Naming Standardization (November 2025)

**Critical Change - Action Required by July 1, 2026**

Microsoft Sentinel is standardizing account entity identification in incidents and alerts with new priority logic:

1. **UPN prefix** (primary): The part before "@" in User Principal Name
   - Example: `john.doe@contoso.com` â†’ `john.doe`
2. **Name** (secondary): Used if UPN prefix unavailable
3. **Display Name** (fallback): Used if both above missing

**Recommended KQL pattern** using coalesce function:
```kusto
coalesce(Account.UPNprefix, Account.Name, Account.DisplayName)
```

**Impact**: May affect analytic rules, automation rules, playbooks, workbooks, hunting queries, and custom integrations

**Required actions**:
- Update KQL queries with new precedence-aware patterns
- Update automation logic
- Update playbooks
- Test in nonproduction workspace before production rollout
- **Deadline: July 1, 2026**
